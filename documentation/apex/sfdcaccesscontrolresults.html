SFDCAccessControlResults

global with sharing virtual class SFDCAccessControlResults
DescriptionThis class provides access control results functionality. This will help to encapsulate and provide more functionality in processing results from access control function calls such as insertAsUser and updateAsUser.

SFDCAccessControlResults.AccessControlResultsException

global class AccessControlResultsException extends Exception

SFDCAccessControlResults.InsertResults

global with sharing class InsertResults extends SFDCAccessControlResults
DescriptionThis class provides results info and functionality for insert operations.

Methods

  getInsertedObjects

global SObject [] getInsertedObjects()
DescriptionGet the objects that were actually inserted into the db.
Note that these objects might not be the same as the objects you provided to the insertAsUser function.
Depending on the current user permissions, the operation mode, and the fields you requested to set, this might
not be all the fields you have in your original objects.

  getResults

global Database.SaveResult [] getResults()
DescriptionGet the Database.SaveResult [] returned by the insert operation.

  InsertResults

global InsertResults(SObject [] objects, Database.SaveResult [] results)
DescriptionThis constructor sets the objects that were actually inserted into the db, and also sets the results from that insert operation.

  wasSuccessful

global Boolean wasSuccessful()
DescriptionDid the insert operation succeed for all objects in the array?
Note that if array operation mode was set to BEST_EFFORT, we will not get an exception even if some
of the objects fail to insert.
