SFDCValidator

global with sharing class SFDCValidator
DescriptionThis class provides basic validation functionality for different types of input.
Note : we don't do any decoding before the checks because that really depends on the context used. Instead, we enforce strict input validation on the input as is.
Note : we expose almost every function in this class in two forms isValid and getValid. They work the same way internally. The only difference is that, the isValid will call the getValid internally, catch any exceptions, and return false in that case. So you can use which ever you prefer, it is only a style preference. the GetValid will always return the exact string as the input.

  SFDC_getValidRedirectLocation

global String SFDC_getValidRedirectLocation(String input, String currentDomain)
DescriptionReturns the input string after validating it is a valid redirect location string. Invalid input will generate an exception. The default validation only allows redirects to *.salesforce.com, current domain, and relative URLs. It also makes sure the URLs only allows http:// and https:// and is no longer than 512 characters long.

Note : this implementation does not protect against directory traversal.

Example:
 try { redirectStr = ESAPI.validator().SFDC_getValidRedirectLocation(unsafe_text, 'www.mydomain.com'); } catch () { // error handling }
ReturnsThe input string after validating it is a valid redirect location
Parameters
inputredirect location to be validated
currentDomaincurrentDomain to be used. (You should only use this if using ApexPages.CurrentPage() does not work for you) If null, will only allow relative URLS and *.salesforce.com domains.
ThrowsException
